# arttle

PS2「ラクガキ王国」風の 1vs1 オンライン対戦ゲームです。自分で描いた絵からステータスを算出し、WebRTC DataChannel で対戦します。

## 構成

- `/frontend`: Next.js 16 (App Router) + React 19 + Canvas API + PeerJS (P2P シグナリング) + Tailwind CSS
- `/signaling-server`: Node.js + WebSocket シグナリング（ローカル開発用。本番は不要）

## 主な仕様

### マッチング・おえかき

- 6桁ルーム番号を発行し、マッチング前は100秒で期限切れ
- 300秒おえかき（完成ボタンで早期確定、時間切れで自動確定）

### ステータス算出（絵の解析）

描いた絵の Canvas ピクセルを解析し、以下の要素からキャラクターのステータス（HP / PP / 攻撃 / 防御 / 速さ / 回避）を算出します。

- **色傾向によるタイプ判定**: 塗られたピクセルの色相（Hue）を集計し、赤〜橙系は「こうげき型」、緑〜紫系は「まほう型」、彩度・明度が低い色は「ぼうぎょ型」に分類。最多カテゴリが突出している場合はそのタイプ、僅差の場合は「バランス型」になる
  - こうげき型: HP・PP低め / 攻撃力・速さ高め
  - まほう型: 攻撃力・防御力低め / PP・速さ・回避高め
  - ぼうぎょ型: 攻撃力・回避低め / HP・防御力高め
  - バランス型: 補正なし（描き方の差がそのまま反映される）
- **塗り量（volume）**: 塗りつぶし面積のカバー率とストロークの太さから算出し、HP・攻撃力・速さに影響
- **描き込み度（detail）**: 使用した色数とストローク（線）の総距離から算出し、PP・回避に影響
- **防御力**: 塗り量と描き込み度を半々で反映

### バトル

- 5秒制限の同時行動ターン制バトル（時間切れは自動選択）
- 行動カテゴリ: こうげき / まほう（弱まほう・強まほう）/ バリア / チャージ
  - 弱まほう: 最大PPの20%を消費
  - 強まほう: 最大PPの40%を消費
  - チャージ: HP・PPを回復し、次のターンのみダメージを1.5倍にする
- **相性関係**（じゃんけん式）: こうげき ➜ バリア ➜ まほう ➜ こうげき
  - 相性で勝った側が一方的にダメージを与える
  - バリア vs まほう: まほうのダメージをバリア側が反射
  - バリア vs チャージ: バリア側が[自身の防御値 × チャージ倍率 - 相手の防御値 ÷ 2]分のカウンターダメージを与える
  - バリア同士: 双方に衝突ダメージ
  - 相性が同じ/なし（例: こうげき同士）の場合は速さが高い方から先制攻撃
- 弱まほうがヒットすると「バリア禁止」「チャージ禁止」「まひ（次ターン行動不可）」のいずれかの追加効果をランダムで付与
- 前ターンと同カテゴリの行動は選択不可、PP不足のまほうは選択不可
- ターン経過による与ダメージ倍率上昇（16ターン以降2倍、21ターン以降3倍）で長期戦を防止
- 回避判定はキャラクターの回避率に基づき確率で発生
- P2P切断時は30秒猶予後に敗北扱い

## 遊び方（Vercel デプロイ済みの場合）

1. **URL を開く** → `https://vsarttle.vercel.app/`
2. **ニックネームを入力 → 「ルーム作成」ボタン**
3. **表示された 6 桁のルーム番号を友達に教える**
4. **友達も同じ URL でルーム番号を入力 → 「入室」ボタン**

> シグナリングサーバーの設定は一切不要です。[PeerJS](https://peerjs.com/) の無料クラウドサーバー経由で P2P 接続を確立します。

## ローカル開発

```bash
cd frontend
npm install
npm run dev
```

ブラウザで `http://localhost:3000` を開き、別タブで同ルーム番号に参加すると 1vs1 対戦できます。

> **注意:** 別タブ間の WebRTC DataChannel は同一ブラウザでは動作しない場合があります。別ブラウザまたは別デバイスで試してください。

### シグナリングサーバー（オプション・ローカルのみ）

```bash
cd signaling-server
npm install
npm run dev
```

デフォルト `ws://localhost:8080` で起動します。フロントエンドからは環境変数で切り替え可能ですが、本番環境では不要です。

## テスト / 検証

```bash
cd frontend
npm run test
npm run lint
npm run build
```

```bash
cd signaling-server
npm run build
```

## デプロイ（Vercel）

1. [Vercel](https://vercel.com/) でリポジトリをインポートする
2. **Project Settings > General > Root Directory** に **`frontend`** を入力して「Save」をクリックする
   > ここを設定しないと "No Next.js version detected" エラーが発生します
3. 「Deployments」タブから「Redeploy」を実行する（または `main` ブランチへ push する）

> **環境変数は一切不要**。`NEXT_PUBLIC_SIGNALING_SERVER_URL` を設定しなくても動作します。
